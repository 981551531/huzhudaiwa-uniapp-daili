# 互助带娃系统 - 视觉风格与配色方案设计

**文档版本**：V1.0
**作者**：Manus AI
**日期**：2026年04月09日

互助带娃系统是一个连接家长、主队/协队（领队）与区域代理商的综合性服务平台。其核心业务涵盖周边游、研学、社区互助等儿童活动。基于其业务属性，系统的视觉设计需要兼顾**亲和力、信任感与专业性**。由于用户端（C端）与代理端（B端）的使用场景和目标人群存在显著差异，本方案为两端分别设计了独立的视觉风格与配色体系。

---
 
## 2. 代理端（B端）视觉风格设计

代理端主要面向区域代理商，使用场景为产品发布、订单管理与数据统计。设计目标是强调**专业、高效、严谨**，帮助代理商快速处理业务，清晰掌握运营数据。

### 2.1 核心设计理念

*   **专业与严谨**：采用偏冷的科技蓝为主色调，传递B端管理工具的专业性与逻辑性。
*   **高效与聚焦**：减少不必要的装饰元素，通过高对比度的色彩和清晰的表格/表单布局，引导用户聚焦于核心数据与操作。
*   **状态分明**：对订单状态、产品审核状态使用明确的色彩标识，便于代理商快速筛查异常情况。

### 2.2 配色方案

代理端的配色以“科技蓝”为主色调，整体色彩饱和度适中，避免长时间使用带来的视觉疲劳。

| 颜色类型 | 颜色名称 | 色值 (HEX) | 使用场景说明 |
| :--- | :--- | :--- | :--- |
| **主色调** | 科技蓝 | `#1890FF` | 全局主按钮（如“保存”、“提交审核”）、TabBar 选中状态、数据看板核心指标、选中边框。 |
| **辅助色 1** | 成功绿 | `#52C41A` | 审核通过状态、已上架状态、订单已完成状态、数据增长趋势（如+15%）。 |
| **辅助色 2** | 警告黄 | `#FAAD14` | 待审核状态、待处理订单、库存预警、数据下降趋势。 |
| **警示色** | 危险红 | `#F5222D` | 审核驳回状态、已下架状态、删除/禁用操作、错误提示。 |
| **背景色** | 商务灰 | `#F0F2F5` | 全局页面背景色，提供沉稳的视觉基调。 |
| **边框色** | 浅边框 | `#E8E8E8` | 表格边框、表单输入框边框、分割线。 |

### 2.3 字体与排版规范

*   **主标题/数据**：字号 `32rpx` - `48rpx`（数据看板数字可更大），字重 `Medium/Bold`，颜色 `#262626`（极深灰），用于页面标题、核心统计数据。
*   **正文/表单**：字号 `28rpx`，字重 `Regular`，颜色 `#595959`（深灰），用于表单标签、表格内容、列表正文。
*   **辅助说明**：字号 `24rpx`，字重 `Regular`，颜色 `#8C8C8C`（中灰），用于表单占位符（Placeholder）、次要说明文字。
*   **圆角规范**：卡片、按钮统一采用 `8rpx` - `12rpx` 的小圆角设计，显得更加干练、规整。

## 3. 交互动效与反馈规范（双端通用）

为提升系统的整体质感，两端在交互反馈上需遵循统一的规范：

1.  **按压反馈**：所有可点击的按钮、列表项（Cell）必须配置 `hover-class`。用户端建议使用浅灰色遮罩（如 `rgba(0,0,0,0.05)`），代理端建议使用主色调的加深色或浅灰色遮罩。
2.  **加载状态**：
    *   **局部加载**：按钮点击提交时，按钮内部需显示 Loading 动画，并禁用点击，防止重复提交。
    *   **全局加载**：页面首次进入或下拉刷新时，优先使用骨架屏（Skeleton Screen）过渡；若无骨架屏，则使用居中的 Loading 图标。
3.  **弹窗与提示**：
    *   **轻提示（Toast）**：用于操作成功（如“保存成功”）、简单的表单校验失败提示，显示 1.5s - 2s 后自动消失。
    *   **确认弹窗（Modal）**：用于破坏性操作（如删除孩子、下架产品、退出登录），必须包含明确的标题、说明文字以及“取消”和“确认”按钮。
4.  **页面切换**：遵循微信小程序原生的右侧推入（Push）和向右滑出（Pop）动画，保持流畅的导航体验。
